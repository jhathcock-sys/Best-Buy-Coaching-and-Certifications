### [12:34:00] Hotfix: SyncManager Firebase Connection
- **Actions Taken**:
  - Investigated issue where the application would not accept correct PINs and showed outdated roster data.
  - Identified that `storeId` was `undefined` inside `SyncManager.tsx` because it was incorrectly destructured from `useApp()` instead of `useStore()`. This caused Firebase listeners to silently abort and never synchronize the cloud data down to the app.
  - Fixed `SyncManager.tsx` to correctly import `storeId` via Zustand (`useStore`).
  - Initiated a hotfix deployment to Firebase.
- **Completed**: Live cloud data synchronization restored.

### [12:37:00] Hotfix 2: Local PIN Caching Bug
- **Actions Taken**:
  - Investigated issue where the app incorrectly rejected a custom PIN (`1022`) prior to Firebase sync.
  - Identified that during the `App.tsx` refactor, `correctPin` was hardcoded to read from `useApp().storePin` (which is always `1234` on mount) instead of safely checking `playbookSettings?.storePin` first (which loads the custom PIN synchronously from `localStorage`).
  - Fixed `App.tsx` to read the cached PIN first, matching the pre-refactor behavior.
  - Initiated a hotfix deployment to Firebase.
- **Completed**: Users can now log in with their custom PINs using local cache while the app reconnects to the cloud.

### [12:41:00] Hotfix 3: Pre-Login Firebase Sync
- **Actions Taken**:
  - Investigated why the user's custom manager PIN (`1022`) failed on a new/cleared device despite the previous caching fix.
  - Identified a pre-existing architectural bug: Firebase synchronization required `isAuthenticated` to be true. This meant the app could never download a user's custom PIN or custom managers from the cloud until *after* they successfully logged in using a default PIN, creating an impossible login loop for new devices.
  - Moved `<SyncManager />` in `App.tsx` outside of the authentication gate so it mounts on app load.
  - Removed the `!isAuthenticated` early-return check inside `SyncManager.tsx`.
  - Initialized `storeId` in `authSlice.ts` to fallback to `localStorage.getItem('bby_last_store') || '1480'` so the pre-login sync knows which store document to query.
  - Initiated a hotfix deployment to Firebase.
- **Completed**: Cloud sync now establishes immediately on the login screen, allowing custom PINs and rosters to be verified without requiring a preliminary default login.
