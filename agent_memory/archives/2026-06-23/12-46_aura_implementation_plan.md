# Goal: Implement Aura (The Live Floor Command HUD)

Build a predictive, tablet-optimized Heads-Up Display (HUD) that actively surfaces "Action Cards" for Floor Leaders. Instead of hunting through metrics, Aura analyzes employee performance and provides AI-generated micro-coaching prompts wrapped in a premium, glassmorphic UI.

## User Review Required

> [!WARNING]
> **API Rate Limiting Consideration**
> If you have a large roster (e.g., 50+ employees) active in a single period, generating AI prompts for every single employee simultaneously on page-load will hit Gemini API rate limits. 
> To prevent this, the plan implements a "Scan Floor" mechanism. The page will load the layout and calculate raw mathematical targets instantly. The AI coaching insights will then populate sequentially (staggered) for the bottom/top quartile performers using our existing `executeWithRetry` backoff function. Let me know if you prefer a different loading strategy.

### AI Engine Integration

#### [NEW] [auraEngine.ts](file:///c:/Users/jhath/projects/Best%20Buy%20Coaching%20and%20Certifications/src/services/ai/auraEngine.ts)
- A dedicated service that consumes `getGeminiModel` and `executeWithRetry`.
- Compiles an employee's raw metrics (Revenue, Apps, Memberships, Transactions) into a short, token-efficient prompt.
- Asks Gemini to return a strict JSON payload containing:
  - `status`: "excellent", "needs_coaching", or "steady"
  - `insight`: A 1-sentence observation.
  - `action`: A 3-4 word recommended action (e.g., "Roleplay Total Tech", "Give High-Five").
